# 更新日志

## v4.7.48

v4.7.48 将 Ideogram4 画布修正下沉到后端规范化路径，并同步 uncen workflow。

- 后端检测到 `__ideogram4_canvas` 时，会用原始画布 shapes 重新生成官方 JSON：对象模块融合为一个场景 `obj`，文字模块保留 `text + bbox`。
- 旧前端或旧历史复用传来的 `Rectangular bounded layout region` / 多对象 bbox JSON 会被兜底重写，避免 uncen workflow 继续画出可见矩形框。
- 文字描述进一步改为“只有字形笔画可见，周围保持原场景”，明确禁止 textbox、caption box、border、半透明填充和容器。

## v4.7.47

v4.7.47 按 Ideogram4 画布的文字/对象分层语义继续修正默认导出。

- 默认照片/场景模式下，所有对象模块只进入一个融合场景 `obj`，不再作为多个带坐标的独立对象元素提交。
- 所有文字模块保留官方 `text + bbox` 精确定位，bbox 只作为排字位置元数据，不生成卡片、边框或分区。
- 聚合对象描述不再混入文字框的位置说明，降低模型把整张图拆成四宫格的倾向，同时保留文字可定位性。

## v4.7.46

v4.7.46 修正 Ideogram4 画布在普通提示词下仍退回四象限 bbox 版式的问题。

- 画布模式现在默认按连续照片/场景融合导出，即使用户没有在总提示词里写“照片/摄影/写实”。
- 只有明确写海报、版式、信息图、平面设计等排版意图时，才启用官方 `bbox` 精确布局路径。
- 默认照片路径继续把画布位置翻译为自然空间语义，避免 upper-left / upper-right / 百分比坐标诱导四宫格。

## v4.7.45

v4.7.45 调整 Ideogram4 照片画布的定位表达，避免 bbox/百分比位置把照片诱导成拼贴或左右分屏。

- 写实/照片模式继续保留画布定位能力，但提交 JSON 时把位置翻译为自然摄影语言，如左侧天空、前景草坪、中心中景，而不是坐标范围。
- 照片模式下对象框聚合为一个连续场景 `obj`，文字框输出为不带 `bbox` 的官方 `text` 元素，用环境文字/标牌语义提高融入感。
- 海报/版式模式仍保留官方 `bbox` 路径，用于需要精确图文排版的场景。

## v4.7.44

v4.7.44 修正 Ideogram4 照片画布仍被模型拆成左右双联、且文字容易丢失的问题。

- 写实/照片模式下，所有对象框先聚合为一个官方 `obj` 场景元素，用完整照片描述承载位置关系，避免多个对象 elements 被渲染成分区画面。
- 文字框保留为官方 `text` 元素并保留 bbox，仅作为 typography placement metadata 使用，以提高文字出现率和位置稳定性。
- 海报/版式模式继续逐元素输出 bbox，保持官方精确布局路径。

## v4.7.43

v4.7.43 将 Ideogram4 画布导出改为照片/版式混合策略。

- 写实/照片提示词下，画布框只用于采集位置和局部提示词；最终官方 JSON 省略硬 `bbox`，把相对位置范围写入元素 `desc`，减少模型生成分屏、双联图和竖向 seam 的倾向。
- 海报、图文版式和非照片提示词仍保留官方 `bbox`，用于更精确的空间布局控制。
- 画布导出的元素描述继续保持官方 `type` / `text` / `desc` key 顺序，并保留具体百分比位置语义。

## v4.7.42

v4.7.42 继续收紧 Ideogram4 画布 JSON 的单张照片语义。

- 画布导出的官方 JSON 会明确要求单张 uninterrupted continuous photograph / single-frame image。
- 全局描述和元素描述新增 split-screen、diptych、vertical divider、gutter、seam 排除，避免模型把多个 bbox 区域渲染成左右分屏。
- 元素 desc 保持官方 bbox 坐标参数，同时强调同一相机视图、同一环境关系和非 side-by-side panel。

## v4.7.41

v4.7.41 按官方 Ideogram4 JSON caption 样本修正布局画布导出语义。

- 画布框只作为“位置 + 局部提示词”的输入编辑器；提交时仍输出官方 `high_level_description`、`style_description`、`compositional_deconstruction.elements` 结构。
- 元素继续保留官方 `bbox` 坐标参数，但 `desc` 改为描述该区域里实际要出现的视觉内容和空间关系，不再把框、区域或 guide 当成画面语义。
- 后端规范化旧 JSON 时会把短描述 + bbox 补成上左/上中/下方等具体位置语义，并保持官方 key 顺序、文本元素和调色板清洗。

## v4.7.40

v4.7.40 调整 Ideogram4 画布在照片/写实场景下的提交策略。

- 当总提示词明显是照片、写实或完整连续场景时，画布对象不再作为硬 `bbox` 元素提交，改为写入相对位置提示，避免 Ideogram4 把区域画成 inset panel。
- 海报、图文版式、非照片场景仍保留官方 bbox 元素路径，用于需要精确布局的提示。
- 文本/对象元素描述会携带上左、上右、下左、下右等相对方位和百分比范围，但不把定位框当作画面内容。

## v4.7.39

v4.7.39 继续收紧 Ideogram4 布局画布的连续场景语义。

- 元素描述不再提及 `bbox`/`inside bbox`，避免模型把定位框理解成画面中的相框或白色卡片。
- 画布 prompt 现在显式排除 picture-in-picture、inset crop、white card、bordered crop、cutout window 等常见拼贴表现。
- 后端旧 JSON 的 `shape` 兜底也改为“隐形 placement/grouping guide”，不再暗示可见边界。

## v4.7.38

v4.7.38 修正 Ideogram4 布局画布过度拼贴化的问题。

- 画布元素不再被描述为可见的矩形/圆形边界区域，改为隐藏定位引导，降低模型画出黑框、分栏和 PPT 拼贴的倾向。
- 画布 JSON prompt 会额外强调同一相机、同一透视、同一光照、同一尺度关系，促使元素融合成一个连续场景或完整照片。
- 后端清洗手写/旧版 Ideogram4 JSON 时也会把 `shape` 解释为隐形构图脚印，而不是要求绘制可见边框。

## v4.7.27
