# 更新日志

## v4.4.0

v4.4.0 聚焦图生图结果核对、远端实例恢复和运行状态表达，让生成后的检查、更换实例、卡住恢复和视频/图片混合任务都更稳。

### 图片对比

- 图生图历史在图片放大窗口左侧新增“图片对比”图标按钮。
- 点击按钮可在当前大图位置切换原图与新图，再次点击返回新图。
- 对比按钮与下载、分享、收藏保持同一套圆形工具按钮视觉，并使用左侧实心的 compare 图标。
- 仅对可找到原始输入图的图生图历史显示，普通文生图、视频和生成中任务不显示。

### 运行恢复

- 增强远端实例停止后的任务收敛，实例被停止时会结束对应残留任务并推动队列继续调度。
- 冷启动等待超时前会先强制重启一次实例，减少旧 ComfyUI 进程占住端口导致的启动假死。
- 新增 GPU 无活动监测，结合进度、节点、显存和 GPU 利用率判断长时间卡住的任务，并按限制重试。
- 实例健康检查会避开仍有活跃任务的实例，减少运行中误判和不必要重启。

### 状态与任务卡片

- 任务状态文案从“图片”改为更中性的“内容/结果”，视频、图片和混合输出使用同一套表达。
- 前端历史卡片和 job 卡片会归一化旧状态消息，减少“视频任务显示拉取图片”的错位感。

### 工作流

- 新增 LTX2.3 10Eros 视频工作流，并登记到工作流元数据中。
- Qwen Edit v2511 暴露更多高级参数，方便在工作流编辑器里调整。

## v4.3.0

v4.3.0 是从 v4.2.x 升级到正式可用版本的一次体验整理，重点放在登录持久化、通知发布、首页入口、工作流扩展，以及历史画廊的载入和刷新稳定性。

### 登录与账号体验

- 登录状态改为更长时间保留，正常情况下至少可维持一个月，减少频繁重新登录。
- 网站标题旁新增当前版本号，方便确认正在使用的版本。
- 账号相关入口继续保持在主界面内，不影响原来的工作流和出图操作。

### 网站通知

- 管理员可在网站内发送通知，只需要填写标题和内容。
- 每条通知都会自动保存发送记录，方便后续查看已经发布过的内容。
- 普通用户进入界面后会以浮窗看到通知。
- 多条通知可以点击“下一条”依次查看。
- 点击关闭只关闭本次浮窗，下次进入仍会显示。
- 点击“不再通知”后，旧通知不再弹出；当管理员发布新通知后，会重新提醒用户。

### 首页入口

- 新增登录前首页，用更简洁的方式介绍系统能力。
- 首页包含大标题、系统说明、工作流/作品等数据块、随机作品展示，以及功能介绍区域。
- 视觉语言保持当前系统配色，同时减少复杂装饰，让进入系统前的第一屏更清晰。
- 点击登录后进入原来的主工作界面，原有生成、历史、工作流管理流程保持不变。

### 工作流与模型

- 新增和整理 Flux.2 Dev / Flux.2 Klein 的文生图、图生图工作流。
- 新增 Qwen Rapid 相关工作流，包括轻量化和 SeedVR2 放大组合。
- 新增 SeedVR2 视频放大工作流，支持 2K / 4K 场景。
- 新增 LTX / Sulphur 视频生成相关工作流。
- 工作流元数据、缩略图、分类和前端展示进一步同步，减少“文件已存在但界面不可见”的情况。

### 生成与历史画廊

- 优化出图历史的懒加载和卡片稳定性，减少首次进入时一次性加载过多内容。
- 生成中的任务卡片改为局部更新，状态变化时不再轻易重绘整块历史区域。
- 管理员视角下的任务轮询改为先比对差异；没有真实变化时只同步数据，不刷新整个历史画廊。
- 修复 admin 页面中出图历史大约每 3 秒整体闪烁一次的问题。
- 优化启动阶段的工作流和历史加载顺序，减少重复请求，让页面载入更快。
- 修复页面初始化时重复触发历史/工作流加载的问题。

### 设备管理

- 优化设备管理在桌面大分辨率下的卡片宽度和布局。
- 保留手机端舒适的单列展示，同时让桌面端设备信息不再被压缩得过小。

### 图像保护与安全展示

- 出图后先进入“图片校验中”状态，由本地轻量校验写回保护状态后再显示。
- 历史卡片使用后端返回的保护状态展示，减少前端重复判断造成的不一致。
- 敏感图片预览、普通用户可见范围、管理员操作能力继续保持后端权限约束。

### 提示词与辅助能力

- 提示词助手支持图片反推、提示词优化和中英切换等能力。
- JSON 提示词的中英文切换更稳定。
- 快速出图和节点编辑中，提示词类字段识别更完整，减少“高级编辑器里有、快速出图里没有”的情况。

### 运行与恢复

- 冷启动、实例选择、任务恢复和取消逻辑继续增强。
- 手动删除或取消的任务不会再被恢复逻辑重新拉回历史卡片。
- 生成计时更贴近真实出图阶段，减少排队、启动、提交阶段对耗时判断的干扰。
- 启动脚本的 restart 流程修复，网站服务重启更稳定。

- 修复 `quick-start.sh restart` 相关流程。
- 增强提示词助手、状态栏、历史画廊和工作流管理的多处细节。
