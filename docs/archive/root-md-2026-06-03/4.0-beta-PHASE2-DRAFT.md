# 4.0-beta Phase 2 — 架构重构与体验优化（讨论稿）

## 需解决的问题

### Issue 1: 全屏闪烁
**问题**：点击启动/停止实例时，整个设备列表重新渲染，导致闪烁。
**根因**：`CW.startInstance/stopInstance` 调用后都执行 `loadNodes()` 全量刷新。
**方案**：改为局部更新——点击后只更新该实例行的状态，不刷新整个列表。

### Issue 2: 工作流同步/上传/目录管理分离
**当前**：同步按钮、上传区、目录管理全部堆在工作流管理面板底部。
**目标**：
- 同步区 → 只显示同步操作（选择设备、同步按钮、同步状态）
- 上传区 → 只做上传
- 目录管理 → 改为「工作流仓库」配置

### Issue 3: 工作流目录重设计

```
/base_dir/                          ← 用户配置的基础目录
├── {device_name}/                  ← 远程同步的工作流（按设备）
│   ├── t2i-z-image.json
│   └── i2i-Qwen-Edit.json
├── ez-comfy/                       ← 手动上传的工作流
│   ├── my-custom-wf.json
│   └── another-wf.json
└── ...其他目录按需...
```

- 设备同步时自动创建 `{device_name}/` 子目录
- 手动上传自动放入 `ez-comfy/`
- 工作流管理中可选择查看：全部 / 某设备 / 手动上传

### Issue 4: 主界面工作流分类
**方向探讨**：
- 按 tag（文生图/图生图/视频）— 现有逻辑
- 按设备（DGX Spark / Mac mini / 手动上传）
- 两者结合：先按 tag 分组，每组内标注来源设备

需要你确认方向。

### Issue 5: 出图历史本地数据库
**方案**：SQLite 存储历史记录，替代当前的 jobs.json
**表结构**：
```sql
CREATE TABLE generation_history (
  id TEXT PRIMARY KEY,           -- 任务ID
  workflow TEXT,                 -- 工作流文件名
  workflow_name TEXT,            -- 工作流显示名称
  device TEXT,                   -- 执行设备
  instance TEXT,                 -- 执行实例
  status TEXT,                   -- done/error
  image_path TEXT,               -- 本地图片路径
  thumb_path TEXT,               -- 缩略图路径
  created_at TIMESTAMP,          -- 创建时间
  completed_at TIMESTAMP,        -- 完成时间
  duration_sec INTEGER,          -- 耗时
  params TEXT,                   -- JSON 参数字段
  prompt TEXT,                   -- 用户提示词
  width INTEGER,                 -- 图片宽
  height INTEGER,                -- 图片高
  seed INTEGER                   -- 种子
);
```
