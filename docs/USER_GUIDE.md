## 1. 安装与启动

### 1.1 环境准备

**必需：**
- Python 3.8+ 运行环境
- NVIDIA DGX Spark (GB10) 或其他 ComfyUI 可运行的 GPU 设备
- Mac 或 Linux 操作系统（用于运行管理服务）

**可选（用于提示词工具）：**
- llama.cpp（`brew install llama.cpp`）
- 视觉语言模型文件（如 Gemma-4-E4B GGUF）

### 1.2 安装项目

```bash
# 克隆仓库
git clone https://github.com/sjcta/ez-comfyui-showcase.git
cd ez-comfyui-showcase

# 安装 Python 依赖
pip install fastapi uvicorn aiofiles pillow python-jose bcrypt
```

### 1.3 启动主服务

**方式一：直接运行**

```bash
# 默认端口 18000
python3 app.py

# 指定端口
python3 app.py --port 9091
```

**方式二：注册为系统服务（推荐）**

```bash
# 启动服务（开机自启 + 崩溃自动重启）
./quick-start.sh start

# 查看服务状态
./quick-start.sh status

# 查看日志
./quick-start.sh logs

# 重启服务
./quick-start.sh restart

# 停止服务
./quick-start.sh stop
```

服务启动后，在浏览器中访问 `http://127.0.0.1:18000/`。

### 1.4 启动 LLM 服务（可选）

如果需要提示词优化、翻译和图片反推功能，需要启动 LLM 服务：

```bash
# 启动 LLM 服务（默认使用 Gemma-4-E4B 视觉模型）
./scripts/mac-llm.sh start

# 查看状态
./scripts/mac-llm.sh status

# 查看日志
./scripts/mac-llm.sh logs

# 停止
./scripts/mac-llm.sh stop
```

LLM 服务默认配置：
- 地址：`http://127.0.0.1:18080`
- 模型：Gemma-4-E4B-Uncensored (Q4 量化)
- 上下文窗口：8192 tokens
- 支持视觉输入（需要 mmproj 投影器文件）

如需切换模型，可通过环境变量配置：

```bash
# 指定自定义模型路径
export EZ_MAC_LLM_MODEL_PATH=/path/to/your/model.gguf
export EZ_MAC_LLM_MMPROJ_PATH=/path/to/your/mmproj.gguf
./scripts/mac-llm.sh restart
```

### 1.5 首次访问

1. 打开浏览器，访问 `http://127.0.0.1:18000/`
2. 你会看到登录前首页，展示系统能力介绍和随机作品
3. 点击 **登录** 按钮进入登录/注册界面
4. 首次使用需要先 **注册账号**
5. 注册完成后自动登录，进入主工作界面

> **提示**：第一个注册的用户默普通用户。如需管理员权限，需要联系现有管理员在用户管理中提升你的角色。

### 1.6 配置设备

首次登录后需要配置 ComfyUI 设备：

1. 点击右上角 **设备管理** 按钮
2. 点击 **添加设备**
3. 填写设备信息：
   - **设备名称**：如 "DGX Spark"
   - **主机地址**：DGX Spark 的 IP 地址（如 `10.10.10.75`）
   - **连接方式**：选择 "SSH - 本地网络"
   - **SSH 用户/端口/密码**：用于远程控制设备
4. 在右侧 **实例预设** 中填写 ComfyUI 端口（默认 `8190,8189`）
5. 填写 **工作流目录**（远程设备上的工作流 JSON 目录）
6. 点击 **保存设备**
7. 系统会自动扫描端口并登记 ComfyUI 实例

```
│  │ 选择 │  │  │               │  │  │ (可吸附)  │   │
│  ├──────┤  │  │  瀑布流卡片   │  │  │           │   │

└──────────────────────────────────────────────────────┘
```

### 2.2 标题栏

| 按钮 | 功能 |
|------|------|
| **设备管理** | 打开设备/节点管理面板，管理 DGX 设备和 ComfyUI 实例 |
| **工作流管理** | 打开工作流管理面板，同步/上传/编辑工作流 |
| **日志** | 打开/关闭日志面板，查看实时运行日志 |
| **用户区域** | 登录/注册入口，登录后显示用户名和退出按钮 |

### 2.3 底部状态栏

| 指标 | 说明 |
|------|------|
| **VRAM** | GPU 显存使用量，进度条可视化（绿/橙/红） |
| **GPU 温度** | 当前 GPU 温度（摄氏度） |
| **GPU 利用率** | GPU 使用率百分比 |
| **ComfyUI 指示器** | 点击打开实例管理弹窗，显示 A/B 实例状态 |

ComfyUI 指示器颜色含义：
- **绿色**：实例待机中
- **橙色/红色**：实例正在生成（随 GPU 压力变化）
- **灰色**：实例未运行

### 2.4 拖拽分隔条

左列和中列之间有可拖拽的分隔条，可以左右拖动调整两列的宽度比例。

## 3. 工作流管理

### 3.1 打开工作流管理

点击标题栏的 **工作流管理** 按钮，打开全屏覆盖层。

### 3.2 同步工作流

1. 在工作流管理面板顶部，选择目标设备
2. 点击 **手动同步** 按钮
3. 系统会从远程 ComfyUI 实例拉取工作流 JSON 文件到本地
4. 同步完成后，工作流卡片自动刷新

### 3.3 上传工作流

**方式一：点击上传**
1. 在工作流管理面板左侧，点击 **上传工作流** 按钮
2. 选择本地的 workflow JSON 文件（支持多选）

**方式二：拖拽上传**
1. 将 workflow JSON 文件直接拖入上传区域
2. 系统自动解析并添加到工作流列表

### 3.4 编辑工作流

点击工作流卡片上的编辑按钮，打开编辑弹窗：

- **名称**：修改工作流显示名称
- **标签**：添加/删除标签（如"文生图"、"4K"），支持从建议列表选择
- **缩略图**：点击上传自定义缩略图（建议横向封面）
- **版本管理**：查看历史版本、上传新版本、切换激活版本

编辑完成后点击 **保存**。

### 3.5 节点编辑器

节点编辑器用于调整工作流中哪些参数在前端显示：

1. 在工作流卡片上点击节点编辑按钮
2. 编辑器显示四个区域：
   - **用户输入**：首屏直接显示的关键参数
   - **高级参数**：折叠在"详细参数"中的进阶控制
   - **输出**：输出相关设置
   - **隐藏**：不在快速出图中显示的字段
3. **拖拽字段卡片**到目标区域即可调整分区
4. 点击眼睛图标切换字段可见性
5. 修改标签文案可改变前台显示名称
6. 点击 **保存配置** 应用更改，或 **恢复默认** 重置

### 3.6 工作流搜索与筛选

- 在工具栏搜索框输入关键词，实时过滤工作流
- 点击分类标签（如"文生图"、"图生图"）筛选特定类型

### 3.7 下载与删除

- **下载**：点击工作流卡片上的下载按钮，保存 JSON 到本地
- **删除**：点击删除按钮，确认后删除工作流及所有版本（不可恢复）

## 4. 文生图 (T2I)

### 4.1 选择工作流

在左列的工作流选择区域，点击选择一个文生图工作流（如 FLUX.2 Dev、Z-Image Turbo）。

### 4.2 输入提示词

选中工作流后，下方会展开快速出图表单：

1. 在 **提示词** 输入框中输入你想生成的画面描述
   - 支持中文或英文
   - 描述越详细，生成效果越好
   - 例如："一只橘猫坐在窗台上，阳光透过窗帘洒落，温暖色调，胶片摄影风格"

### 4.3 设置参数

**基础参数**（首屏可见）：
- **尺寸**：选择预设分辨率或自定义宽高
- **提示词**：正向和负向提示词

**详细参数**（点击"详细参数"展开）：
- **随机种子**：固定种子可复现相同结果，留空随机
- **采样步数**：步数越多细节越好，但耗时更长
- **CFG**：提示词引导强度
- **其他**：取决于工作流配置

### 4.4 提交生成

1. 确认参数无误后，点击 **开始生成** 按钮
2. 任务进入队列，中列画廊区域显示任务卡片
3. 卡片会实时显示进度条和预计剩余时间
4. 生成完成后，卡片自动更新为结果预览图

### 4.5 查看结果

- 点击结果卡片打开 Lightbox 大图查看
- 可下载原图、收藏、分享、导出到其他工作流
- 点击左右箭头浏览上一张/下一张

## 5. 图生图 (I2I)

### 5.1 选择工作流

选择一个图生图工作流（如 Qwen Edit、FireRed Edit）。

### 5.2 上传参考图

1. 在快速出图表单中，找到图片上传区域
2. 点击上传或拖拽一张参考图片
3. 图片会同步到远程 ComfyUI 实例

### 5.3 配置编辑指令

- 在提示词中描述你希望如何修改参考图
- 例如："将背景改为海边日落场景，保持人物不变"
- 调整降噪强度（denoise）控制修改幅度

### 5.4 生成与对比

1. 点击 **开始生成**
2. 完成后在 Lightbox 中可查看图片对比：
   - 点击 **对比按钮**（compare 图标）可在原图和新图之间切换
   - 再次点击返回新图

### 5.5 导出到下游

在 Lightbox 中，可将结果导出到：
- **图生图**：继续编辑
- **视频制作**：以当前图片为起始帧生成视频
- **放大**：进行超分辨率放大

## 6. 视频生成 (I2V)

### 6.1 选择工作流

选择一个视频生成工作流（如 LTX 2.3 Sulphur、LTX 2.3 10Eros）。

### 6.2 输入图片与脚本

1. 上传起始帧图片
2. 在提示词中描述视频运动和内容
3. 系统会自动将描述编排为视频脚本（包含时间轴）

### 6.3 导演模式（10Eros Director）

如果使用导演版工作流，可以点击 **导演模式** 按钮展开高级编辑区：

1. 上传多张参考图
2. 为每个片段设置：
   - 起始帧和长度
   - 运动强度
   - 片段提示词
3. 系统会将多段编排写入 `timeline_data`

### 6.4 查看视频

生成完成后：
- 在 Lightbox 中点击播放按钮观看视频
- 使用时间轴拖拽逐帧预览
- 点击 **设为封面** 可设置视频缩略图
- 导出按钮可将视频帧发送到其他工作流

## 7. 超分辨率放大

### 7.1 选择工作流

选择一个放大工作流（如 SeedVR2 2K、SeedVR2 4K）。

### 7.2 上传低分辨率图

上传需要放大的图片。

### 7.3 生成

点击 **开始生成**，等待放大完成。放大任务通常需要较长时间（4K 约 2 分钟）。

### 7.4 查看与对比

放大完成后，可在 Lightbox 中查看放大效果。如果是图生图的放大结果，可使用对比按钮查看原图与放大后的差异。

## 8. 批量生成

### 8.1 设置批量参数

在快速出图表单中，可以设置批量生成参数：
- **批量数量**：一次生成多张
- 每张使用不同的随机种子

### 8.2 批次管理

- 批量任务会依次进入队列
- 画廊中同一批次的任务会关联显示

## 9. 历史画廊

### 9.1 浏览历史

中列显示瀑布流形式的出图历史：
- 初始加载 9 张卡片
- 向下滚动自动加载更多
- 每张卡片显示预览图、工作流名称、参数摘要

### 9.2 搜索与筛选

- **搜索框**：输入关键词搜索提示词内容
- **归属筛选**：
  - 所有图片：显示全部历史
  - 我的图片：仅显示当前用户的生成结果
  - 我的收藏：仅显示已收藏的内容
  - 其他：其他用户的内容
- **类型筛选**：按图片/视频类型筛选

### 9.3 Lightbox 查看

点击卡片打开 Lightbox：

- **图片**：大图查看，支持缩放
- **视频**：视频播放器，支持逐帧预览
- **导航**：左右箭头或键盘方向键切换上一张/下一张
- **底部信息**：显示工作流、参数、耗时等详细信息

Lightbox 工具栏：

| 按钮 | 功能 |
|------|------|
| 收藏 | 收藏/取消收藏当前内容 |
| 分享到首页 | 公开/取消公开 |
| 隐藏 | 隐藏内容（管理员可见） |
| 导出 | 导出到图生图/视频制作/放大 |
| 下载 | 下载原图/视频 |
| 对比 | 图生图历史可切换原图/新图 |
| 视频编辑 | 视频可逐帧预览和设置封面 |
| 删除 | 删除当前记录 |

### 9.4 删除与恢复

- **删除**：在 Lightbox 或卡片上点击删除，记录进入回收站
- **恢复**：在回收站中可以恢复已删除的记录
- **永久删除**：彻底删除记录和文件，不可恢复
- **清空回收站**：批量永久删除所有回收站内容

### 9.5 快速复用

在历史卡片上，可以点击 **复用** 按钮，一键将历史参数回填到快速出图表单，修改后重新生成。

## 10. 提示词工具

### 10.1 提示词优化

1. 在快速出图表单中，找到提示词助手区域
2. 输入你的简单描述
3. 点击 **优化** 按钮
4. 系统会通过 LLM 将描述扩展为专业级提示词
5. 优化结果可直接回填到提示词输入框

优化能力：
- 自动补充画面细节（光线、材质、氛围）
- 适配当前选择的模型特性
- 支持视频脚本优化（LTX/Seedance 风格）

### 10.2 中英互译

- **中译英**：将中文提示词翻译为英文（适合主流 AI 模型）
- **英译中**：将英文提示词翻译为中文（方便理解）
- 支持 JSON 格式提示词的语言切换

### 10.3 图片反推

上传一张图片，AI 自动分析并输出可复用的提示词。

**操作步骤：**

1. 在提示词助手区域点击 **图片反推** 按钮
2. 上传要分析的图片（截图、照片均可）
3. 选择反推模式：

| 模式 | 特点 | 推荐场景 |
|------|------|----------|
| **标准反推** | 快速输出基础画面描述 | 想快速获取大致描述 |
| **专家反推** | 按维度拆解，结构化 JSON 输出 | 需要高精度复刻 |
| **专家团反推** | 9 位专家协同分析，最精细 | 追求极致还原度 |

4. 等待分析完成
5. 查看结果，可以：
   - **复刻完整**：将完整提示词回填到出图输入框
   - **复刻正向**：仅回填正向提示词
   - **复刻负面**：仅回填负面提示词（写入独立负面框）

> **提示**：专家团反推需要较长时间（约 1-2 分钟），因为需要进行多轮分析。

## 11. 设备管理

### 11.1 查看设备列表

点击标题栏 **设备管理** 按钮，打开设备管理面板。

面板显示所有已注册的设备卡片，包含：
- 设备名称和主机地址
- 连接状态
- 实例列表和状态
- 标签信息

### 11.2 添加设备

1. 点击 **添加设备** 按钮
2. 填写设备信息：

**设备身份：**
- 设备名称（如 "DGX Spark"）
- 主机地址（IP 或域名）
- 连接方式（SSH/HTTP/本机）
- 标签（如 "GPU,高性能"）
- 共享设置（是否允许其他用户使用）

**SSH 连接配置**（远程设备需要）：
- SSH 用户名
- SSH 端口（默认 22）
- 认证方式（密码/密钥）
- 密码或密钥路径

**访问入口：**
- 访问 URL 模板（如 `https://example.com:{port}/`）

**实例预设：**
- 默认端口列表（如 `8190,8189`）
- 工作流目录路径

3. 点击 **保存设备**

### 11.3 管理实例

在设备卡片上可以操作每个 ComfyUI 实例：

| 操作 | 说明 |
|------|------|
| **启动** | 启动 ComfyUI 服务 |
| **停止** | 停止 ComfyUI 服务 |

## 14. 常见问题 (FAQ)

### Q: 实例启动失败怎么办？

**A:** 按以下步骤排查：
1. 在设备管理中点击 **测试连接**，检查 SSH 和 HTTP 连通性
2. 确认远程设备上 ComfyUI 的 systemd 服务已注册
3. 尝试 **强制重启** 实例
4. 查看日志面板中的错误信息
5. SSH 登录远程设备，手动执行 `systemctl --user status comfyui-a` 检查服务状态

### Q: GPU 显存不足怎么办？

**A:**
1. 检查底部状态栏的 VRAM 使用量
2. 停止不必要的 GPU 进程（在设备管理中查看 GPU 进程列表）
3. 确保出图前 vLLM 服务已停止（系统通常会自动处理）
4. 尝试使用较小的分辨率或更轻量的模型

### Q: 任务一直排队不执行？

**A:**
1. 检查是否有正在执行的任务（全局串行队列，同时只能有一个任务执行）
2. 检查 ComfyUI 实例是否正常运行
3. 如果实例卡住，尝试重启实例
4. 系统会自动检测卡死任务并进行纠错（最多 3 次重试）

### Q: 工作流同步失败？

**A:**
1. 确认远程设备上的 ComfyUI 实例正在运行
2. 确认设备配置中的工作流目录路径正确
3. 确认 SSH 连接正常
4. 在设备管理中点击 **测试连接** 排查

### Q: 提示词工具没有反应？

**A:**
1. 确认 LLM 服务已启动：`./scripts/mac-llm.sh status`
2. 在系统设置中检查 LLM API 地址是否正确
3. 点击 **测试连接** 验证连通性
4. 检查 LLM 服务日志：`./scripts/mac-llm.sh logs`

### Q: 图片反推结果质量不好？

**A:**
1. 尝试使用 **专家反推** 或 **专家团反推** 模式，精度更高
2. 确保上传的图片清晰、分辨率足够
3. 确认 LLM 使用的是视觉模型（带 mmproj 投影器）
4. 专家团模式需要更长的处理时间，请耐心等待

### Q: 页面刷新后任务状态丢失？

**A:**
- 正在执行的任务会在页面刷新后自动恢复显示
- 失败和取消的任务也会持久化保留
- 如果仍有问题，检查 `data/jobs.json` 文件是否正常写入

### Q: 如何在 DGX Spark 上部署更新？

**A:**
```bash
# 在 DGX 上执行部署脚本
ssh sjcta@10.10.10.75 "bash /home/sjcta/deploy.sh"
```

部署脚本会自动拉取最新代码、切换 main 分支并重启服务。

### Q: 如何切换 LLM 模型？

**A:**
1. 将新的 GGUF 模型文件放入 `models/llm/` 目录
2. 设置环境变量指向新模型：
   ```bash
   export EZ_MAC_LLM_MODEL_PATH=/path/to/new/model.gguf
   export EZ_MAC_LLM_MMPROJ_PATH=/path/to/new/mmproj.gguf
   ```
3. 重启 LLM 服务：`./scripts/mac-llm.sh restart`
4. 在系统设置中更新模型名称并测试连接

### Q: 浏览器支持情况？

**A:**

- 支持 Safari（已做兼容性处理）
- 支持移动端浏览器（响应式布局）
- 建议使用 HTTPS 访问（通过 Nginx 代理）
