# Phase 1 - 远程工作流同步

## 概述
支持从远程设备同步 ComfyUI 工作流到本地，方便集中管理和展示。

## 连接方式
- local → 本地文件系统直接读取
- remote-ssh → SSH 远程读取
- remote-http → HTTP 从 ComfyUI API 拉取

## 需要修改的文件
1. app.py
2. static/js/modules/workflows.js
3. static/js/modules/nodes.js（可能需要）
4. static/index.html（可能需要）

## 具体实现

### 后端 app.py
1. 设备配置文件：当前 config/nodes.json 中的设备配置，需要增加 `workflow_dirs` 字段（可选，远程路径列表）
2. 新增 `sync_remote_workflows()` 函数：
   - 遍历所有 enabled 设备
   - connection=local：直接扫描本地 WORKFLOW_DIR 目录
   - connection=remote-ssh：SSH find + cat
   - connection=remote-http：HTTP GET
   - 解析 workflow json 提取元数据
   - 更新本地 data/wf_meta.json 和 data/wf_dirs.json
3. 在 lifespan（启动时）和每 5 分钟调用一次 sync_remote_workflows()
4. 确保本地 data/workflows/ 目录有缓存的工作流 json 文件

### 前端 workflows.js
1. 现有 loadWorkflows/loadWfMeta 已从本地文件读取
2. 可能需要增加加载状态："同步远程工作流中..."

### SSH 实现
- 使用 sshpass -p '{password}' ssh ...
- 对每个返回的文件路径：sshpass cat 读取
- 解析 json，写入本地 data/workflows/{basename}
- 更新 wf_meta.json

### HTTP 实现
- Phase 1 简化：远程 HTTP 设备暂不支持自动同步

### 约束
- 不要破坏已有功能
- 同步后自动刷新前端列表
